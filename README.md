# архив

Каркас статичного сайта-каталога для промптов и наработок.
Работает на чистом HTML/CSS/JS — без сборки, без зависимостей.
Можно просто открыть `index.html` в браузере или выложить на GitHub Pages.

## Структура

```
index.html        — главная
catalogue.html    — каталог (сетка карточек, фильтры, поиск)
assets/style.css  — стили (палитра и шрифты)
assets/app.js     — логика + НАСТРОЙКИ (имя сайта, ник, ссылки)
data/prompts.js   — ТВОИ ПРОМПТЫ (правишь в основном этот файл)
```

## Как добавить промпт

Открой `data/prompts.js` и допиши блок в список:

```js
{
  title: "Название карточки",
  category: "Соло",                 // фильтр создастся сам
  tags: ["neon", "rain"],           // для поиска, можно []
  prompt: "сам текст промпта",      // его копирует кнопка
  note: "комментарий",              // можно ""
  image: "assets/img/example.jpg"   // путь к примеру, можно ""
}
```

Новые категории появляются автоматически из поля `category`.
Картинки положи в `assets/img/` и укажи путь в поле `image`.

## Как поменять имя, ник и ссылки

В начале файла `assets/app.js` — блок `CONFIG`:

```js
const CONFIG = {
  siteName: "архив",
  tagline:  "промпты и наработки",
  handle:   "@yourname",
  links: { telegram: "", github: "", profile: "" }
};
```

## Выложить на GitHub Pages

1. Создай новый репозиторий на GitHub (например `archive`).
2. Загрузи в него все эти файлы (кнопка **Add file → Upload files**,
   либо `git push`). Файлы должны лежать в корне репозитория.
3. Зайди в **Settings → Pages**.
4. В разделе **Build and deployment** выбери
   **Source: Deploy from a branch**, ветка `main`, папка `/ (root)`,
   нажми **Save**.
5. Через минуту сайт будет доступен по адресу
   `https://ТВОЙ-НИК.github.io/archive/`.

Чтобы обновить сайт — просто заливай изменённые файлы в репозиторий,
Pages пересоберётся сам.

## Цвета и шрифты

Палитра и шрифты задаются переменными в начале `assets/style.css`
(блок `:root`). Меняешь там одно значение — обновляется везде.
